## 3. 引擎抽象层设计

### LLM Provider 接口

```python
class LLMProvider(ABC):
    @abstractmethod
    def chat(self, system: str, user: str, **kwargs) -> str: ...
    @abstractmethod
    def chat_json(self, system: str, user: str, **kwargs) -> dict: ...
    @abstractmethod
    def chat_stream(self, system: str, user: str) -> Iterator[str]: ...
```

| Provider | API | 成本 | 适用场景 |
|----------|-----|------|----------|
| DeepSeek | chat.deepseek.com | ~¥0.5/M tokens | 默认，低成本首选 |
| Qwen (DashScope) | dashscope.aliyuncs.com | ~¥1/M tokens | 国内稳定备选 |
| GLM-4 (Zhipu) | open.bigmodel.cn | ~¥1/M tokens | 长上下文场景 |
| GPT-4o | api.openai.com | ~$5/M tokens | 高质量高复杂度 |
| Claude Sonnet | api.anthropic.com | ~$3/M tokens | 创意写作优化 |
| 本地 LLM | localhost | 免费 | 完全离线 |

### Image Provider 接口

```python
class ImageProvider(ABC):
    @abstractmethod
    def generate(self, prompt: str, n: int = 1, size: str = "1920x1080") -> list[str]: ...
    @abstractmethod
    def download(self, url: str, output_path: Path) -> Path: ...
```

# 第四部分：关键设计决策

## 架构原则

1. **抽象层先于实现** — 每个 AI 能力都先定义接口，再写具体适配
2. **本地优先** — 能本地跑的绝不依赖 API (TTS/图像/SD 都支持本地)
3. **渐进式增强** — Phase 1 先稳 MVP，绝不跳过基础直接做花哨功能
4. **成本透明** — 每个 Project 记录 API 调用次数和预估成本
5. **可插拔** — 所有 Provider 注册制，新增一行代码即可接入新模型

## 技术选型说明

| 选型 | 理由 |
|------|------|
| SQLite (不换 PG) | 单机场景够用，零运维，数据简单 |
| Gradio (不换 React) | MVP 改造成本最低，Phase 4 再考虑换 |
| ffmpeg (不换专业 NLE) | 功能足够，零依赖，命令行可控 |
| pip + venv | 简单可靠，用户基础最广 |
| Pydantic v2 | 类型安全，JSON 序列化原生支持 |
